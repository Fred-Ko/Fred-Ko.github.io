# 확장 프로그램 테스트 가이드

## 의심스러운 확장 프로그램

목록을 보면 직접적으로 코드 하이라이팅과 관련된 확장은 없지만, 다음 확장들이 페이지 콘텐츠를 수정할 수 있습니다:

### 1. **AdGuard 광고차단기** (가장 의심스러움)
- 광고 차단기는 때때로 페이지 스크립트를 수정합니다
- 코드 블록을 변환할 가능성은 낮지만, 스크립트 로딩을 방해할 수 있습니다

### 2. **우클릭 차단 해제**
- 페이지의 JavaScript를 수정하여 우클릭 차단을 해제합니다
- 코드 블록을 변환할 가능성은 낮지만, 스크립트 실행 순서를 변경할 수 있습니다

## 테스트 방법

### 단계별 테스트:

1. **모든 확장 프로그램 비활성화**
   - `chrome://extensions/` 또는 `edge://extensions/` 접속
   - 모든 확장 프로그램 토글 OFF
   - 페이지 새로고침 후 `.token` 클래스 확인

2. **하나씩 활성화하며 테스트**
   - 가장 의심스러운 것부터:
     - AdGuard 광고차단기
     - 우클릭 차단 해제
   - 각각 활성화 후 페이지 새로고침하여 확인

3. **브라우저 콘솔에서 확인**
   ```javascript
   // 확장 프로그램이 주입한 스크립트 확인
   Array.from(document.querySelectorAll('script'))
     .filter(s => s.src.includes('extension'))
     .forEach(s => console.log(s.src));

   // .token 클래스 확인
   console.log('.token 개수:', document.querySelectorAll('.token').length);
   ```

## 빠른 확인 방법

시크릿 모드에서 페이지 열기:
- 확장 프로그램이 자동으로 비활성화됨
- `.token` 클래스가 사라지면 확장 프로그램이 원인
